### Contribution Process

1. **Fork the repository**
   ```bash
   git clone https://github.com/your-username/awesome-3d-prompts.git
   cd awesome-3d-prompts
   ```

2. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow the [prompt format guidelines](#prompt-format)
   - Ensure your prompts are high-quality and tested
   - Add prompts to the appropriate category directory

4. **Commit your changes**
   ```bash
   git add .
   git commit -m "Add: description of your contribution"
   ```

5. **Push and create a Pull Request**
   ```bash
   git push origin feature/your-feature-name
   ```

### Prompt Format

Each prompt file should follow this structure:

```markdown
# Title

**Category**: [category name]
**Style**: [style description]
**Tool**: [recommended tool(s)]
**Tags**: tag1, tag2, tag3

## Prompt

[The actual prompt text]

## Parameters

- Parameter 1: value
- Parameter 2: value

## Description

[Additional notes and descriptions]

## Variations

- Variation 1
- Variation 2

## Example Use Cases

- Use case 1
- Use case 2

---

**Author**: Your Name or Community
**Last Updated**: YYYY-MM-DD
```

### File Naming

- Use kebab-case: `my-awesome-prompt.md`
- Be descriptive: `fantasy-warrior.md` not `prompt1.md`
- Include category in complex names: `modern-office-chair.md`

### Guidelines

1. **Quality over quantity** - Only submit prompts you've tested
2. **Be descriptive** - Include parameters, variations, and use cases
3. **Follow structure** - Use the standard format for consistency
4. **Add tags** - Help others find your prompts
5. **Test before submitting** - Ensure prompts work with recommended tools

### Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and grow
- Follow the project's guidelines
